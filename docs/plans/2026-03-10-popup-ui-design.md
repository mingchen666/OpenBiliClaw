# Popup UI Refresh Design

## Background

`extension/popup/` 已经具备推荐、画像、聊天三类功能，但当前界面仍然偏“深色工具面板”：

- 视觉重心放在后端连接状态，而不是推荐内容本身
- 三个 tab 共享逻辑，但页面气质和信息层级还不够统一
- 推荐卡片、画像 chips、聊天输入区的视觉语言偏硬，缺少 B 站内容产品常见的轻快感

用户希望 popup 更美观，并尽量贴近 Bilibili 的明亮、活力、内容导向风格。

## Goal

在不改变 popup 现有功能行为的前提下，完成一次纯前端展示优化：

- 改为明亮活力的浅色主题
- 强化推荐内容的吸引力和点击欲望
- 统一推荐 / 画像 / 聊天三页的视觉系统
- 保持 popup 小尺寸下的清晰层级和可读性

## Non-Goals

- 不新增后端 API
- 不改推荐、反馈、画像、聊天的数据结构
- 不引入新的前端框架或 UI 库
- 不修改 extension manifest 或构建链

## Visual Direction

采用 **B 站轻活力型**：

- 主背景为暖白 + 浅粉蓝渐变，避免当前深色工具感
- 品牌强调色采用 B 站语境接近的 `#FB7299`，辅以 `#5AA9FF`
- 卡片统一使用大圆角、浅阴影、柔和边框
- 内容优先级从“系统状态”切换为“推荐内容”
- 交互动效仅保留轻量颜色变化、阴影变化和 1px 位移

## Layout

popup 保持三页签结构，但重构顶部和内容区：

1. **Hero Header**
   - 展示产品名和一句简短价值文案
   - 右侧放轻量状态徽标，弱化技术信息占比

2. **Segmented Tabs**
   - 以胶囊式 tab 替换现有硬边分栏按钮
   - 选中态突出，但不过度抢主内容

3. **Content Surface**
   - 三个 tab 内统一使用浅色卡片系统
   - 推荐页优先进入内容列表
   - 画像页先展示主画像摘要，再展示 chip 分组
   - 聊天页统一消息卡与输入卡视觉语言

4. **Feedback Footer**
   - 保留底部提示文本，但视觉存在感降级
   - 仅用于状态反馈和用户操作结果回显

## Recommendation View

推荐卡片层级固定为：

1. 顶部标签带：主题标签、推荐状态
2. 标题：两行内成为视觉主角
3. 辅助说明：UP 主与推荐表达
4. 动作区：主按钮“立即打开”，次按钮“先跳过”“写一句”

交互要求：

- 整卡可点击，按钮操作不会误触发打开视频
- hover / focus-visible 要明显但克制
- 空状态改为更柔和、更像“内容整理中”的提示样式

## Profile View

- 顶部保留一张主画像卡，承载人格摘要
- `核心特质 / 深层需求 / 当前偏好` 改为统一信息卡
- chip 使用低饱和浅色块，而不是当前单一灰蓝色
- 未初始化状态使用与推荐空状态一致的留白风格

## Chat View

- 聊天气泡区改为浅色消息气泡
- 阿B 回复与用户消息颜色分层清楚
- 输入区改为更像提问卡片的容器，发送按钮拥有明确主操作视觉
- 保持现有消息 append 行为和滚动到底逻辑

## Accessibility

- 保留现有 `role="tablist"` / `tabpanel` 语义
- 增强键盘聚焦样式
- 轻色主题下确保正文与辅助文字仍满足可读性
- 使用 `prefers-reduced-motion` 降低动态效果

## Testing Strategy

由于当前 extension 没有 popup DOM 测试基建，本次测试继续分两层：

1. 纯逻辑测试
   - 为 popup helper 补充与视觉状态相关的最小测试
   - 锁定 tab state、empty state、fallback 文案等稳定行为

2. 构建与回归验证
   - `npm test`
   - `npm run typecheck`
   - `npm run build`

3. 手动验证
   - 启动本地后端后打开 popup
   - 检查浅色主题、tab 切换、推荐卡片、画像卡、聊天输入区是否正常显示
   - 验证推荐打开、反馈输入、聊天发送无行为回归

## Acceptance

- popup 首屏观感明显比当前版本更亮、更轻、更贴近 Bilibili 内容产品气质
- 推荐内容成为首要视觉焦点
- 推荐、画像、聊天三页风格统一
- 现有交互行为保持稳定，无明显回归
