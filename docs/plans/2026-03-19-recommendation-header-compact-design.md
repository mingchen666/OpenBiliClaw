# Recommendation Header Compact Design

当前推荐页头部已经从散开的三段结构收成了一张卡，但对于内容型入口来说，它仍然占了太多首屏面积：

- `For You`、标题、说明文字占了完整一层
- 三块状态又各自占了一整排卡片高度
- 第一张推荐卡被明显往下推，内容主角感不够

这次优化的目标不是减少信息，而是把头部从“信息面板”进一步收成“紧凑推荐头条”。

## 目标

1. 明显缩短推荐头部高度，让更多推荐内容进入首屏
2. 保留 `当前可换 / 最近补进 / 现在在忙` 三条状态
3. 保留 `换一批` 的主操作优先级
4. 继续维持当前浅色、轻玻璃、内容产品化的气质，不做后台工具栏

## 方案

采用“紧凑双层头部”：

1. 第一层：标题行
   - `For You` 从独立一行改成更小的 inline kicker
   - 标题与 kicker 同行组织
   - `换一批` 固定在右侧
2. 第二层：状态 chips 行
   - 保留三条状态
   - 从三张独立状态卡改成三枚轻量 chips
   - 每条状态用 `标签 + 短文案` 的一行结构表达

说明文字继续保留，但弱化为更短的二级说明，不再撑出完整“导语区”。

## 视觉语言

- 整个头部仍然保留一张统一容器卡，但 padding 明显缩小
- 标题区改成更紧的编辑式排版，强调内容入口而不是运营面板
- `For You` 使用更像导航标识的轻量 kicker，而不是一整行视觉锚点
- 三个状态块改成圆角 chips：
  - `当前可换` 偏中性
  - `最近补进` 偏浅粉
  - `现在在忙` 偏浅蓝
- chips 采用更轻的边框和更短的行高，避免视觉体积过重
- 长状态文案优先压缩成更短的 copy，必要时用省略显示

参考 `ui-ux-pro-max` 设计系统：

- 模式偏 `content-first`
- 风格偏 `Swiss Modernism 2.0`
- 原则是更强的阅读效率、更数学化的间距、更轻的装饰层

## 文案策略

为了适配 chip 形式，状态文案收短：

- `当前池子里还有 151 条可换` -> `还有 151 条可换`
- `刚补进 6 条新的` -> `刚补进 6 条`
- `这会儿先不补货，池子里已经够你换了` -> `这会儿先不补货`
- `最近在补：...` -> 直接显示更短的实时内容本体

保留语义，但去掉对 chip 无帮助的冗长前缀。

## 交互规则

- `换一批` 仍然在头部第一视觉层
- loading 时按钮文案仍可切换为 `正在换一批…`
- `现在在忙` chip 继续承接 runtime / loading 文案
- chip 内容允许在窄宽度换行，但默认优先单行显示
- 窄宽度下 chips 允许折成两行或三行，不再强制三列卡片

## 实现边界

本次只收缩推荐头部，不改：

- 推荐卡片结构
- 反馈按钮流转
- 推荐接口和 runtime 协议
- footer activity card

允许调整：

- 头部 markup 结构
- pool status 的展示 class 和布局方式
- pool status copy 的长度和语气

## 验证口径

自动化需要保证：

1. 头部不再使用大状态卡网格
2. 头部存在紧凑标题层和 chips 层
3. pool status copy 收成更适合 chip 的短文案
4. loading / runtime 覆盖逻辑不回归

手动验证需要确认：

1. 第一张推荐卡明显更早进入首屏
2. 头部仍然美观，不像开发态筛选条
3. `换一批`、库存信息、运行状态都能快速扫读
4. 窄宽度下 chips 不拥挤、不炸版
