# For You Magazine Hero Design

## 背景

`For You` 推荐区已经从松散面板收成了更紧凑的头部和更规整的卡片，但整体仍然偏“组件卡片列表”：

- 推荐首屏缺少真正的视觉主角
- 第一张内容和后续内容的权重没有被拉开
- 头部虽然已经变轻，但内容区仍然更像后台卡片，不像内容产品入口

用户这轮明确希望改成更偏“杂志感”的内容呈现，而不是继续打磨当前的工具型卡片。

## 目标

1. 让第一张推荐成为真正的主视觉封面卡
2. 保持头部简洁，不再往上增加说明文字
3. 后续推荐继续保持规整，但要被主卡自然压住视觉权重
4. 不改 popup 数据协议和交互逻辑，只做前端呈现升级

## 方案

采用“主视觉封面卡 + 常规列表卡”的杂志式结构：

### 1. 首卡成为封面卡

- 第一张推荐卡切成纵向内容块，不再和后续卡片共用同一密度
- 封面明显放大，作为内容入口的第一视觉层
- 标题更像文章标题，字级更大，换成更强的 headline 节奏
- 标签和状态缩成更细的眉标，放在标题附近，而不是和按钮同等显眼

### 2. 操作区编辑化

- `去看看 / 多来点 / 少来点 / 说说原因` 仍然保留
- 但按钮体量与排布更像内容卡页脚工具栏，不再像功能控制台
- 第一张卡的操作区和内容宽度统一，避免碎

### 3. 列表层级

- 第一张卡明显大于后续卡片
- 第二张及以后仍然维持当前较紧凑的规整卡片语言
- 整个推荐流形成“头部导语 -> 封面主卡 -> 常规卡列表”的节奏

## 实现边界

本次允许调整：

- `extension/popup/popup.html` 中推荐卡样式与首卡特化 CSS
- 必要的 popup 渲染 class
- 布局测试
- extension 文档与 changelog

本次不改：

- 推荐排序逻辑
- 推荐数据结构
- 反馈接口和行为
- 画像与聊天视图

## 验证口径

自动化需保证：

1. 首张推荐卡存在明显区别于普通卡片的 hero 样式
2. 普通卡片仍保持可读和响应式稳定
3. `For You` 头部继续维持当前紧凑结构，没有副说明文案回归

手动期望：

1. 首张卡一眼看上去像杂志封面，而不是普通列表项
2. 推荐流层级更像内容产品，不像后台面板
3. 杂志感来自排版和版式，不靠更重的渐变和装饰
