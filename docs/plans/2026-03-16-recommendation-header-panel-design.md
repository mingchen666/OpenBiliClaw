# Recommendation Header Panel Design

当前 popup 推荐页的头部信息被拆成了三段平铺内容：

- `view-intro` 负责标题和说明
- `recommendation-toolbar` 负责 `换一批`
- `pool-status` 负责候选池和运行时状态

问题不在于信息量过多，而在于这些信息没有被组织成稳定层级。标题、主操作、实时状态和库存状态现在都处于同一视觉平面，读起来更像“说明文字 + 日志块”，而不是推荐入口。

这次优化只做推荐区头部的信息架构整理，不改推荐卡片本身，也不改后端状态逻辑。

## 目标

1. 让推荐页头部更像一个轻量信息面板，而不是零散堆叠的段落
2. 强化主操作 `换一批` 和“当前可换数量”的可读性
3. 保留运行时动态信息，但避免整个区块像后台监控面板
4. 让 loading / success / idle / enough-stock 几种状态切换时不跳布局

## 方案

采用“推荐头部卡 + 三段状态面板”的结构。

### 结构调整

把当前的：

- `view-intro`
- `recommendation-toolbar`
- `pool-status`

合并成一个新的推荐头部卡。

头部卡分成两层：

1. 上层：标题信息 + 主操作
   - 左侧放 `For You`、标题、说明
   - 右侧放 `换一批`
2. 下层：三块语义明确的小状态面板
   - `当前可换`
   - `最近补进`
   - `现在在忙`

### 视觉语言

整体保留 popup 现有的浅色玻璃卡体系和粉蓝品牌气质，但状态块本身改得更克制、更清楚：

- 标题区继续走内容产品语气，强调“这是推荐入口”
- 状态区改成卡内卡结构，每块都有：
  - 小标签
  - 主文案
- `当前可换` 使用偏中性的浅底，突出库存信息但不喧宾夺主
- `最近补进` 使用非常轻的粉色语义底，延续补货和推荐的品牌感
- `现在在忙` 使用浅蓝语义底，让运行态像信息提示，不像报警
- 文本对比度高于当前版本，避免浅灰文字糊成一片

有意不做的事：

- 不引入更强的渐变或发光效果
- 不做 dashboard 式密集网格
- 不加跳动、闪烁、轮播类动画
- 不把实时状态做成全局主视觉

## 交互规则

状态区的目标是“明确，但不打断内容浏览”。

### 默认态

- 三块状态面板都展示稳定内容
- 不轮播、不自动切换
- 面板高度固定，避免不同状态切换时引起整体跳动

### 点击 `换一批`

- 主按钮进入 loading 态
- 按钮文案仍可切换成 `正在换一批…`
- 原来漂浮在按钮旁边的 `refreshRecommendationsStatus` 不再作为独立一行展示
- “现在在忙” 面板优先显示 `正在给你换一批…`

### 请求成功

- `当前可换` 立即刷新为新的库存数字
- `最近补进` 保持最近一次补货结果，或显示“这会儿先不补货，池子里已经够你换了”
- `现在在忙` 回退到实时运行文案或默认 topic 文案

### 请求失败

- 不把整张头部卡切成错误态
- 只在“现在在忙”这块里显示短错误文案
- 这样可以避免整个推荐入口看起来像故障面板

### 运行时消息优先级

- 如果 websocket / runtime stream 有即时 `message`，优先占用“现在在忙”
- 没有即时消息时，再回退到默认 topic 文案

## 实现边界

本次只改推荐头部区域：

- 重组 `view-intro`
- 重组 `recommendation-toolbar`
- 重组 `pool-status`

明确不改：

- `recommendation-list` 渲染结构
- 推荐卡片内容、按钮和反馈流转
- API 返回结构
- 后端 runtime / pool 文案生成逻辑

允许的轻微前端行为变化：

- `refreshRecommendationsStatus` 可以被弱化或并入状态面板
- 推荐头部可能新增更语义化的容器 class 和状态块 class

## 验证口径

自动化覆盖需要保证以下几件事：

1. popup 布局测试能锁住新的头部卡结构
2. popup copy 测试继续保证核心文案存在
3. helper 测试覆盖：
   - 默认 pool summary
   - enough-stock 文案
   - runtime 消息覆盖默认 topic
4. loading 态不会依赖独立漂浮状态行才能传达“正在换一批”

手动验证关注：

- 推荐 tab 默认打开时，头部层级是否清楚
- 点击 `换一批` 后是否没有明显 layout jump
- enough-stock / 补货中 / 换一批中 / 错误态是否都能一眼读懂
- 窄宽度下三块状态面板是否还能稳定换行而不拥挤
