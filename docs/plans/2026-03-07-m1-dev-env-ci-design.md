# M1 开发环境与 CI 设计

**目标**

完成 `docs/v0.1-todolist.md` 中 `1.1 开发环境和 CI` 的最小可交付版本，让仓库具备稳定的本地开发入口和基础持续集成检查。

**范围**

- 验证并修复 `pip install -e ".[dev]"` 安装链路
- 让 `ruff check src/ tests/`、`mypy src/`、`pytest` 在本地可运行
- 为仓库新增 GitHub Actions 工作流，至少执行 lint、type check、tests
- 仅修复为达成上述目标所需的最小代码或配置问题

**不在范围内**

- 不实现新的业务功能
- 不推进 `1.2 配置系统完善` 之后的任务
- 不重构模块结构或引入新框架

**实现策略**

先以当前代码为基线建立开发环境，再逐个运行质量门禁，记录真实失败项。修复顺序按“安装问题 -> 静态检查问题 -> 测试问题 -> CI 固化”推进，确保每一步都有可重复验证的命令。

**风险与边界**

- 当前本地环境尚未安装 `pytest`，说明必须从依赖安装开始
- 若某个失败暴露出 todo 范围外的大缺口，仅做让 `1.1` 可成立的最小修复
- CI 配置将与本地验证命令保持一致，避免出现“本地过、CI 不过”的双重标准

**验收标准**

- 新克隆仓库后可执行 `pip install -e ".[dev]"`
- 本地通过 `ruff check src/ tests/`、`mypy src/`、`pytest`
- 新增 `.github/workflows/ci.yml`，并覆盖 lint、type check、tests
