Soul 模块负责「理解用户」。下面把它的几条主要流程拆成步骤讲清楚。英文术语首次出现时用 等宽蓝字 标出并就地加中文解释，完整对照见 §7。一句话总览：各种信号（行为 / 反馈 / 点击 / 对话）汇入一条流水线，按「该影响哪几层、攒够没、间隔到没」分层更新五层画像；同时两个探针主动猜测你的新兴趣与想避开的方向。

A. 一条信号怎么变成画像更新（主流水线）
  1. 入口：行为事件 / 推荐反馈 / 推荐点击 / 对话都先变成「信号」，交给 ProfileUpdatePipeline（画像更新流水线）的 ingest()。行为事件要等画像初始化完成（is_profile_ready()）后才收，初始化期间（约 7 分钟）先跳过。
  2. 分类：classify_signal（信号分类）判断这条信号该影响哪几层 —— 例如反馈影响 兴趣 / 表层 / 价值，点击影响 兴趣 / 表层。
  3. 缓冲：信号进各层的 LayerBuffer（分层缓冲区）排队；同时顺手喂给两个探针的 observe()（旁观匹配，见 D/E）。
  4. 是否够了：每层有门槛 —— 攒够 N 条信号 且 距上次更新够久才更新。但若缓冲里有「强信号」（反馈 / 对话 / 点击），门槛直接降到 1 条、立刻更新（时间间隔仍生效）。
  5. 更新：_update_layer 调对应的 layer_updaters（分层更新器）改写该层；若更新器报错，信号放回缓冲不丢失。
  6. 重写人物旁白：若改的是「核心层」或「价值层」，触发 regenerate_portrait 重新生成 personality_portrait（一段叙事性的人物总述）。
  7. 落盘：有变化时保存画像 JSON / Markdown，并追加一条变更日志。
B. 五层 OnionProfile 各层的更新节奏（内层慢·强证据，外层快）
  1. 核心层 Core（特质 / 深层需求 / MBTI）：攒 8 条 / 间隔 48 小时，大模型「增量」更新 + 最强保护（MBTI 几乎不变）。
  2. 价值层 Values（价值观 / 驱动力）：5 条 / 24 小时，大模型增量。
  3. 兴趣层 Interest（兴趣树 / 喜欢的 UP 主）：3 条 / 10 分钟，由 PreferenceAnalyzer（偏好分析器）重算。
  4. 角色层 Role（人生阶段 / 当前任务）：5 条 / 24 小时，大模型增量。
  5. 表层 Surface（语气 / 内容形态 / 探索开放度）：3 条 / 5 分钟，纯计算、不调大模型。
  6. 「强信号」一律把门槛降到 1 条（间隔仍生效）；只有核心 / 价值层真的变了才会重写人物旁白。内层证据要求高、变得慢，外层贴近当下、变得快。
C. 即时反馈 vs 批量重析
  1. 即时：单条 like / dislike / comment 会立刻写一张「认知卡」（record_immediate_feedback_cognition）—— 轻量记录，不重建画像。
  2. 批量：攒够 3 条新反馈才触发 process_feedback_batch_if_needed，重跑偏好分析；只有判定为「显著变化」（权重变动≥0.2 / ≥2 个高权兴趣变动 / 新增避雷）才重建整份画像。
  3. 对话候选：聊天里提取的候选要 置信度≥0.8 且 出现≥2 次，才会喂进偏好重析。
D. 正向兴趣探针（InterestSpeculator）
  1. 生成：大模型主动猜你可能感兴趣的新方向，分两个池 —— 「近域 near」（最多 5 个）和「挑战池」（lateral 横向 / bridge 桥接 / wildcard 天马行空，最多 3 个）。每 10 分钟一次。
  2. 三条确认路：① 被动命中——你的行为关键词命中探针 3 次，自然晋升；② 强正向——你在聊天 / 卡片里明确表达喜欢 → 直接确认；③ 弱正向——先进「短期探索缓冲」short_term_exploration_buffer，攒够分（≥4.0 且 ≥3 次正向 & ≥3 次显式，7 天内）才晋升。
  3. 晋升：merge_confirmed_interest 把它并入兴趣层，按确认来源给权重（来源越硬权重越高：画像确认 0.60 / 聊天 0.50 / 探针·缓冲 0.45 / 仅猜测 0.30）。
  4. 过期：3 天没被确认就否决并冷却 7 天（聊天里被否决冷却 14 天）。
E. 避雷探针（AvoidanceSpeculator）
  1. 对称但更保守：结构和兴趣探针一样，但只认高确信负向 —— 你明确点了「不喜欢 / 踩」才计数。⚠ 被动快速划走、短停留点击绝不算数，也绝不自动写进长期避雷。
  2. 确认后清池：apply_new_dislikes 把它写进偏好层 disliked_topics、同步画像，并清理候选池 pool_purge（先按字符串硬清，再用向量召回 0.55 + 大模型精判；没有大模型时退回向量兜底 0.78）。
  3. 在避雷探针上点正向 = 直接否决该条避雷（冷却 14 天）。
F. 后台 60 秒 tick 做的三件事

_loop_soul_pipeline 每 60 秒调一次 pipeline.tick()：

  1. 排空缓冲：把各层攒够的信号落下去更新。
  2. 探针生命周期：过期 → 晋升 → 生成（空闲时按 30 分钟节流，避免空转）。
  3. 认知周期 CognitionCycle：每 12 小时一次 —— 先写「观察笔记」（awareness）再提「假设洞察」（insight），写回画像。失败时不推进计时，下个 tick 立刻重试，而不是干等 12 小时。
G. 派生与存储
  1. ToneProfile（语气画像）：读取时从画像现算（密度 / 温度 / 俏皮 / 直接四个维度），不落盘，用来驱动推荐文案和聊天语气。
  2. ProfileOverrides（用户手动编辑）：存在单独文件，读画像时叠加在 AI 画像之上 —— 你删掉的项会一直被压制，不会被 AI 重新生成又覆盖回来。
  3. cognition_updates.json（认知卡）：interest_added / dislike_added / profile_shift 等卡片，是画像变化的可读记录，供插件画像页展示。

OnionProfile当前的五层「洋葱」画像（内→外：核心 / 价值 / 兴趣 / 角色 / 表层）。

cognition_updates认知卡：画像变化的可读记录（interest_added / dislike_added / profile_shift）。
④ 信号与反馈

record_immediate_feedback_cognition单条反馈即时写一张认知卡（不重建画像）。

InterestSpeculator正向兴趣探针：主动猜测你可能感兴趣的新方向。
AvoidanceSpeculator避雷探针：主动猜测你想避开的方向（只认高确信负向）。
